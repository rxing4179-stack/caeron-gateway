# Phase 2.5 改进指令

## 任务概述
对现有管理面板进行 UI/UX 改进，包括中文化、功能增强和配色方案更换。

## 1. 提示词注入表单中文化

将注入规则相关的所有英文选项和标签翻译为中文：

### 角色选项（role）
- `system` → `系统消息`
- `user_wrapped_system` → `用户包装的系统消息`
- `assistant_prefill` → `助手预填充`

### 注入位置选项（position）
- `system_prepend` → `系统消息前置`
- `system_append` → `系统消息追加`
- `dialog_start` → `对话开头`
- `before_latest` → `最新消息之前`
- `at_depth_N` → `指定深度插入`

### 表单标签
- `规则名称` ✓（已是中文）
- `内容` 下面的提示文字保持中文 ✓
- `匹配条件 (JSON)` → 改为 `匹配条件`，并在下方加说明文字：`设定规则的生效条件，留空则对所有请求生效。示例：{"match_model": "gpt-4o", "match_length_min": 5}`
- `优先级 (小=高)` ✓
- 所有按钮文字确保为中文（保存、取消、编辑、删除、启用、禁用）

### 注意
- 下拉选项的显示文字改为中文，但传给后端的 value 保持英文不变
- 即 `<option value="system">系统消息</option>` 这种形式

## 2. 供应商管理 - 模型拉取功能

在"添加供应商"和"编辑供应商"表单中，为"支持的模型"字段增加自动拉取功能：

### 功能需求
1. 在模型输入框旁边增加一个「拉取模型列表」按钮
2. 点击按钮后，使用用户已填写的 API Base URL 和 API Key，向上游供应商发送 `GET /models` 请求（标准 OpenAI 兼容接口）
3. 拉取成功后，弹出一个模型选择面板（弹窗或内嵌展开区域），包含：
   - 搜索框：支持按模型名称实时过滤
   - 模型列表：带复选框，用户可勾选需要的模型
   - 全选/取消全选按钮
   - 确认按钮：将勾选的模型填入输入框（逗号分隔）
4. 如果拉取失败（API地址无效、key错误等），显示友好的中文错误提示
5. 原有的手动输入方式保留，两种方式并存

### 后端支持
在 `main.py` 中添加一个代理端点：
```
POST /admin/api/providers/fetch-models
Body: {"base_url": "https://...", "api_key": "sk-..."}
Response: {"models": ["gpt-4o", "gpt-4o-mini", ...]}
```
这个端点负责代理请求上游的 `/models` 接口，避免前端跨域问题。

## 3. 全局配色方案更换

### 核心要求：从深色主题改为浅色/青绿色主题

### 色卡（由浅到深）
- 最浅背景色：`#deebec`
- 次浅背景色：`#c0dade`
- 中间色/卡片背景：`#aecfd1`
- 强调色/按钮悬停：`#77b5b4`
- 主题色/按钮/高亮：`#4199a0`

### 配色映射规则
- **页面整体背景**：`#deebec`
- **卡片/面板背景**：`#ffffff`（白色），边框用 `#aecfd1`
- **侧边栏/导航栏背景**：`#c0dade`
- **主要按钮（保存、确认等）**：背景 `#4199a0`，文字白色，悬停时 `#77b5b4`
- **危险按钮（删除）**：保持红色系但降低饱和度，与整体协调
- **输入框背景**：`#ffffff`，边框 `#aecfd1`，聚焦时边框 `#4199a0`
- **表格/列表**：白色背景，交替行用 `#deebec`
- **文字颜色**：主文字 `#2c3e50`（深灰蓝），次要文字 `#5a7a7d`
- **标签页（Tab）**：未选中用 `#77b5b4`文字，选中用 `#4199a0` 底色+白字
- **开关（Toggle）**：开启状态 `#4199a0`，关闭状态 `#aecfd1`
- **弹窗遮罩**：半透明的 `rgba(65, 153, 160, 0.3)`

### 注意
- 整体风格从暗黑系改为清新青绿系
- 确保文字在所有背景色上的对比度足够（WCAG AA标准）
- 弹窗表单的背景用白色，与页面背景形成层次
